ARCHITECTURE NOTES
================================================================================

SPEC DESIGN PRINCIPLES:
  1. Each rule is independent and testable
  2. Rules cover critical paths (not all paths — that would be exhaustive)
  3. Invariants are preserved across all state transitions
  4. Properties are behavioral (what should happen, not implementation)
  5. Edge cases explicitly handled (empty arrays, zero amounts, etc.)

FORMAL VERIFICATION APPROACH:
  1. Invariant-based: Critical properties that always hold
  2. Behavioral: Safety properties (what shouldn't happen)
  3. Exhaustive: Check all execution paths (not just test cases)
  4. Automatic: No manual proof needed (SMT solver does it)

KNOWN LIMITATIONS:
  1. External contracts assumed correct (Aave, USDC)
  2. Non-reentrant guard assumed working (not verified)
  3. Pausable mechanism not verified (separate audit needed)
  4. Owner immutability assumed (no owner transfer function)
  5. Array size bounded to reasonable limits (<100 donors)
