### Key Implementation Details

**Immutable Logs:**
- `admin_logs` table has no UPDATE or DELETE in application code
- Only INSERT operations allowed (audit trail integrity)
- Retention cleanup deferred (not in schema, manual job for later)

**Soft Revocation:**
- Roles never deleted, revoked_at timestamp set instead
- Preserves audit trail and FK integrity
- Active roles: `WHERE revoked_at IS NULL`

**Role Scope:**
- Global roles: scope_type=NULL, scope_id=NULL
- Scoped roles: scope_type='church'|'community'|'project', scope_id=UUID
- Hierarchy applies only to global roles (higher privilege users can grant lower roles)

**Health Checks:**
- Append-only time series (rolling samples)
- Latest check per name retrieved via `order('created_at DESC).limit(1)`
- Uptime %= OK checks / total checks in time window
- Overall status logic: down (if any down) > degraded (if any degraded) > ok

**Dashboard Metrics:**
- KPIs: total users + breakdown by role (from user_roles table, where revoked_at IS NULL)
- Health: latest status for all checks + overall status
- Timestamp: current time (metrics computed on-demand)
