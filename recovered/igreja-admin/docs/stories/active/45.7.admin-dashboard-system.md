# Story 45.7: Dashboard Admin — System Administration & Monitoring

**Story ID:** 45.7  
**Depends On:** Stories 45.1-45.3

## 🎯 Acceptance Criteria

### System Health Monitoring
- [x] Dashboard home (/admin/dashboard) with KPIs
- [x] Service status (API, Supabase, Web3, Realtime)
- [x] Uptime tracking (% uptime)
- [x] Health check endpoints (database, external APIs)
- [x] Alert management (critical alerts highlighted)

### User Management
- [x] User list (paginated, searchable, sortable)
- [x] User detail view (profile, activity, roles)
- [x] Assign roles (admin, moderator, donor, church_admin, member)
- [x] Disable/enable user accounts
- [x] User activity log (last login, actions)
- [x] User permissions per feature (Comunidade, Plantar, Semear)

### Financial Reporting
- [x] Total donations (all-time, monthly, by church)
- [x] Donation breakdown (by project, by donor)
- [x] Withdrawal/distribution tracking
- [x] Charts: Donation trend (line), By church (pie), By project (bar)
- [x] Export to CSV
- [x] Date range filtering

### Security & Audit Logs
- [x] Audit trail (all admin actions logged)
- [x] Login history (successful + failed attempts)
- [x] Failed login tracking + account lockout
- [x] Change logs (who changed what, when)
- [x] IP address tracking
- [x] Searchable logs (by user, action, date)
- [x] Log export functionality

### Content Moderation
- [x] Recent posts (threads, comments, ideas)
- [x] Flag/report management
- [x] Content removal workflow
- [x] Moderator assignment
- [x] Moderation history

### System Settings
- [x] Site-wide configuration (name, contact, terms)
- [x] Email template management
- [x] Rate limiting configuration (per endpoint)
- [x] Backup scheduling (show status)
- [x] Maintenance mode toggle

### Analytics & Reports
- [x] User growth chart
- [x] Engagement metrics (posts/month, active users)
- [x] Project completion rate
- [x] Idea approval rate
- [x] Custom date range reports

### React Components
- [x] AdminDashboard (home page)
- [x] HealthStatus (service status)
- [x] UserManagement (table + detail views)
- [x] RoleAssignment (role picker)
- [x] FinancialReports (charts + tables)
- [x] AuditLog (searchable log viewer)
- [x] SystemSettings (configuration forms)
- [x] Analytics (dashboard charts)

### Testing
- [x] Permission-based access (admin-only routes)
- [x] Data integrity (no orphaned records)
- [x] Audit logging (all changes tracked)
- [x] Report generation accuracy
- [x] >70% test coverage

### Performance & UX
- [x] Dashboard loads <1s
- [x] Charts render smoothly
- [x] Large lists paginated (100 per page)
- [x] Search results <500ms
- [x] Lighthouse ≥80

## 📊 API Endpoints

```
GET    /api/admin/dashboard                     — KPIs + health status
GET    /api/admin/health                        — Health check
GET    /api/admin/health/services               — Individual service status

GET    /api/admin/users                         — List users (paginated)
GET    /api/admin/users/[id]                    — Get user detail
PUT    /api/admin/users/[id]/role               — Update user role
PUT    /api/admin/users/[id]/status             — Enable/disable user
GET    /api/admin/users/[id]/activity           — User activity log

GET    /api/admin/reports/financial             — Financial summary
GET    /api/admin/reports/donations             — Detailed donations
GET    /api/admin/reports/by-church             — Donations by church
GET    /api/admin/reports/by-project            — Donations by project
GET    /api/admin/reports/export                — CSV export

GET    /api/admin/logs                          — Audit log (paginated)
GET    /api/admin/logs/login                    — Login history
GET    /api/admin/logs/changes                  — Change log

GET    /api/admin/analytics/growth              — User growth chart
GET    /api/admin/analytics/engagement          — Engagement metrics
GET    /api/admin/analytics/projects            — Project completion rate

PUT    /api/admin/settings                      — Update system settings
GET    /api/admin/settings                      — Get settings
```

---

## 📝 Notes

- All admin actions logged to admin_logs table
- Role-based access (admin role required)
- Audit trail provides compliance documentation
- Financial reports for transparency
- Health monitoring enables proactive incident response
- Real-time status updates via WebSocket (Story 46.1)
