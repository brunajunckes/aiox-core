# Story 8.3: Git Workflow Automation

## Objective

Reduce the friction of Git branching and committing during story development by introducing a structured `aiox git flow` sub-command that automates branch creation from story IDs, enforces naming conventions, and bundles the pre-commit quality gate into a single finish command.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer implementing AIOX stories,
**I want** CLI commands that automate Git branch creation, quality-gate execution, and commit preparation from a story ID,
**so that** branch names are always consistent, quality checks are never skipped, and context switches between stories are frictionless.

## Acceptance Criteria

- [x] `aiox git flow start <story-id>` creates a Git branch named `story/{sprintNum}.{storyNum}-{slug}` from the story ID and checks it out locally
- [x] The slug in the branch name is derived from the story title found in `docs/stories/<story-id>.story.md` (lowercase, words joined by hyphens, max 40 chars)
- [x] `aiox git flow finish` runs `npm test` and `npm run lint` sequentially; if both pass, stages all tracked changes and creates a conventional commit message referencing the current story ID
- [x] The commit message format is: `feat: <story title> [Story <story-id>]`
- [x] `aiox git flow status` shows: current branch, detected story ID, last test result (pass/fail/not-run), and uncommitted file count
- [x] `aiox git flow start` exits with code 1 if the story file does not exist in `docs/stories/`
- [x] `aiox git flow finish` exits with code 1 without committing if `npm test` or `npm run lint` fails; it prints which check failed and the output
- [x] Branch naming convention is `story/{sprintNum}.{storyNum}-{slug}` ‚Äî e.g. `story/8.3-git-workflow-automation`
- [x] Zero external npm dependencies ‚Äî implemented with Node.js built-ins and `child_process`

## Dev Notes

**Architecture Constraint ‚Äî CLI First (Article I):**
All git-flow logic is CLI-only. No UI involved. State file `.aiox/flow-status.json` lives in L4 (always mutable).

**@devops Authority:**
`aiox git flow finish` commits locally ‚Äî it does NOT push. Push authority remains exclusively with @devops (`git push`, `gh pr create`). The finish command only calls `git commit`.

**Slug Generation Algorithm:**
```js
function toSlug(title) {
  return title
    .toLowerCase()
    .replace(/[^a-z0-9\s-]/g, '')
    .trim()
    .replace(/\s+/g, '-')
    .slice(0, 40)
    .replace(/-+$/, '');
}
```

**Story ID Parsing from Branch:**
Branch format: `story/{sprintNum}.{storyNum}-{slug}`. Parse with `/^story\/(\d+\.\d+)-/`.

**Flow Status File:**
`.aiox/flow-status.json` ‚Äî written by `finish` after gate run:
```json
{ "storyId": "8.3", "lastGate": "pass", "timestamp": "<ISO>" }
```

**Relevant Source Tree:**
- CLI router: `bin/aiox.js`
- Story files: `docs/stories/`
- Flow status: `.aiox/flow-status.json`
- Entity registry: `.aiox-core/data/entity-registry.yaml`

**Testing:**
- Test location: `tests/unit/git-flow/`
- Stub `spawnSync` for git and npm commands ‚Äî do not run real git operations in unit tests
- Use fixture story files for resolver tests
- Integration smoke: call `start` against a temp git repo
