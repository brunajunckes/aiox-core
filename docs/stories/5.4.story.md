# Story 5.4: Multi-language Documentation Generator

## Objective

Implement `aiox docs generate --lang <code>` to generate translated documentation from English source files. The MVP produces translated files with `[TRANSLATE: original text]` placeholders for paragraph-level content, preserving all headings, code blocks, and structure exactly. Supports Portuguese (pt), Spanish (es), and Chinese Simplified (zh) â€” matching the existing `docs/` structure.

## Dev Notes

**Architecture Constraint â€” CLI First (Article I):**
`aiox docs generate` is a pure CLI content pipeline. No external translation API calls. MVP is scaffolding only â€” human translators fill placeholders.

**Zero External Dependencies:**
Use only Node.js built-ins: `fs`, `path`, `os`. No markdown parser libraries.

**Line Classifier State Machine:**
```javascript
function classifyLines(content) {
  const lines = content.split('\n');
  let inFence = false;
  let inFrontmatter = false;
  let fmCount = 0;

  return lines.map((line, i) => {
    if (i === 0 && line === '---') { inFrontmatter = true; fmCount++; return { type: 'frontmatter', content: line }; }
    if (inFrontmatter) {
      if (line === '---') { fmCount++; if (fmCount === 2) inFrontmatter = false; }
      return { type: 'frontmatter', content: line };
    }
    if (line.startsWith('```')) { inFence = !inFence; return { type: 'code', content: line }; }
    if (inFence) return { type: 'code', content: line };
    if (/^#{1,6}\s/.test(line)) return { type: 'heading', content: line };
    if (line.trim() === '') return { type: 'blank', content: line };
    return { type: 'prose', content: line };
  });
}
```

**Placeholder Insertion Pattern:**
```javascript
function transform(nodes) {
  return nodes.map(node => {
    if (node.type === 'prose') return `[TRANSLATE: ${node.content}]`;
    return node.content;
  }).join('\n');
}
```

**Skip-Existing Pattern:**
```javascript
if (fs.existsSync(targetPath) && !force) {

  return;
}
