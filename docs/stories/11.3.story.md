# Story 11.3: Notification System

## Objective

Provide a CLI command to manage notifications stored in `.aiox/notifications.jsonl` with read/unread tracking.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** a notification system that queues and displays messages,
**so that** I can receive important updates from CLI operations and other commands.

## Acceptance Criteria

- [x] `aiox notify` shows unread notifications from `.aiox/notifications.jsonl`
- [x] `aiox notify --all` shows all notifications including read
- [x] `aiox notify --mark-read` marks all as read
- [x] `aiox notify --clear` clears all notifications
- [x] `addNotification(title, body, type)` exported utility for other commands
- [x] Types: info, success, warning, error
- [x] Notifications have id, title, body, type, read, timestamp
- [x] Command registered in `bin/aiox.js`
- [x] 15+ tests in `tests/cli/notify.test.js`
