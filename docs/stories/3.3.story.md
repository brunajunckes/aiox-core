## Story
**As a** developer using AIOX across multiple terminal sessions,
**I want** cross-session state carry-over that saves and restores my active story, agent, and branch context,
**so that** I can resume exactly where I left off without reconstructing my working context each time I open a new session.

## Acceptance Criteria

1. - [ ] Session state is persisted to `.aiox/session-state.json` on every state change — file is created if missing
2. - [ ] Persisted state includes: active story ID, active story path, active agent, current git branch, last action timestamp, workflow position
3. - [ ] `aiox resume` command reads `.aiox/session-state.json` and prints a structured summary of the last session state
4. - [ ] `aiox resume` suggests the concrete next action based on the restored state (e.g., "Story 3.1 is InProgress — run `aiox dev` to continue implementation")
5. - [ ] `aiox status` command is enhanced to include session info section: active story, active agent, time since last activity
6. - [ ] Session state survives process restarts (file-based, not in-memory only)
7. - [ ] State write is atomic — use write-to-temp-then-rename pattern to prevent corruption on crash
8. - [ ] `aiox resume` gracefully handles missing or corrupt `.aiox/session-state.json` with a helpful message and clean exit

## Dev Notes

**Builds on Story 2.3:**
- Story 2.3 implemented runtime state tracking and `aiox status`; review `docs/stories/2.3.story.md` for the state module location and the existing `aiox status` implementation
- The `.aiox/handoffs/` directory is already used for agent handoff artifacts — `.aiox/session-state.json` is a sibling file in the same `.aiox/` directory
- Do NOT overwrite or merge with the handoff artifacts — `session-state.json` is a separate, distinct file

**Atomic write pattern (critical for AC7):**
```javascript
const tmpPath = statePath + '.tmp';
fs.writeFileSync(tmpPath, JSON.stringify(state, null, 2), 'utf8');
fs.renameSync(tmpPath, statePath);
```
This prevents partial writes from corrupting the state file on crash.

**Session state JSON schema:**
```json
{
  "activeStoryId": "3.1",
  "activeStoryPath": "docs/stories/3.1.story.md",
  "activeAgent": "dev",
  "gitBranch": "story/3.1-agent-discovery",
  "lastActionTimestamp": "2026-04-08T14:30:00.000Z",
  "workflowPosition": "InProgress"
}
```
The `workflowPosition` field maps to story lifecycle statuses: Draft, Ready, InProgress, InReview, Done.

**Next-action suggestion logic (for `aiox resume`):**
Map `workflowPosition` to a suggested command:
- `Draft` → "Story is in Draft — ask @po to validate: `aiox po validate`"
- `Ready` → "Story is Ready for development — start with: `aiox dev`"
- `InProgress` → "Story is InProgress — continue implementation: `aiox dev`"
- `InReview` → "Story is in QA review — check QA status: `aiox qa`"
- `Done` → "Story is Done — create next story: `aiox sm create-story`"

**`aiox status` backward compatibility:**
- Existing status output (git branch, recent work, current epic) must remain unchanged
- The new "Session" section is appended at the end
- Use a horizontal separator line before the section
