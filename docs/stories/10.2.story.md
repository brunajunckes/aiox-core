# Story 10.2: Command Execution Timer & Performance Logging

## Objective

Provide an `aiox perf` command that tracks CLI command execution times in a JSONL log and displays performance metrics (last N, slowest, averages).

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer optimizing AIOX CLI performance,
**I want** to track and view command execution times,
**so that** I can identify slow commands and measure improvements.

## Acceptance Criteria

- [x] `aiox perf` shows last 10 command execution times from `.aiox/perf-log.jsonl`
- [x] `aiox perf --slow` shows top 5 slowest commands
- [x] `aiox perf --clear` clears the performance log
- [x] `aiox perf --avg` shows average execution time per command
- [x] Each entry format: `{ command, duration_ms, timestamp }`
- [x] Handles missing/empty log file gracefully
- [x] Duration formatted as ms or seconds depending on magnitude
- [x] Table output with aligned columns
- [x] Command registered in bin/aiox.js

## Tasks

- [x] Create command module at `.aiox-core/cli/commands/perf/index.js`

- [x] Implement last-N, slowest-N, and average calculations
