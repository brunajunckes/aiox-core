# Story 6.2: Interactive CLI Wizard

## Objective

Implement `aiox wizard` — a step-by-step interactive project setup wizard that guides the user through project name, template selection, agent configuration, and first story creation. The wizard uses Node.js `readline` exclusively (no inquirer), persists answers to `core-config.yaml`, and can be re-run at any time to update the configuration.

## Agent

@dev (Dex)

## Status

Draft

## Story

**As a** developer onboarding to AIOX for the first time (or reconfiguring an existing project),
**I want** a guided wizard that walks me through project setup step-by-step,
**so that** I can configure AIOX correctly without reading documentation or hand-editing YAML files.

## Acceptance Criteria

- [ ] `aiox wizard` launches an interactive terminal wizard using Node.js `readline` (no inquirer dependency)
- [ ] Wizard presents steps in order: (1) Project Name, (2) Template Selection, (3) Agent Configuration, (4) First Story Creation
- [ ] Step 1 — Project Name: prompts for project name, validates non-empty string, shows current value as default if re-running
- [ ] Step 2 — Template Selection: presents a numbered list of available templates (nextjs-react, go, java, rust, csharp, php); user selects by number; validates input is a valid option number
- [ ] Step 3 — Agent Configuration: prompts user to enable/disable each core agent (dev, qa, architect, pm, po, sm) via y/n; shows current values as defaults if re-running
- [ ] Step 4 — First Story Creation: asks if user wants to create a first story draft now (y/n); if yes, prints instructions to run `aiox sm *draft` after wizard completes (does NOT trigger story creation directly)
- [ ] On completing all steps, wizard writes answers to `core-config.yaml` under a `wizard` section and prints a success summary
- [ ] Re-running `aiox wizard` pre-fills all prompts with current values from `core-config.yaml` so the user can update specific fields by accepting defaults for unchanged ones
- [ ] Wizard can be cancelled at any step by pressing Ctrl+C; on cancel it prints "Wizard cancelled. No changes saved." and exits with code 0 without writing any partial config
- [ ] All prompts display the expected input format (e.g., `[1-6]`, `[y/n]`, `[Enter to keep: current-value]`)
- [ ] Unit tests cover: each step's input validation, default value population from existing config, cancel behavior (no writes), successful completion writing to config

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
The wizard is a CLI-only experience. No web UI, no TUI framework (no blessed/ink). Pure readline.

**Zero New External Dependencies:**
Use only Node.js built-in `readline` module and existing `js-yaml` if already in `package.json`. Do not add inquirer, prompts, or any other interactive input library.

**Readline Pattern:**
```javascript
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

// Promisified question helper
function ask(rl, question) {
  return new Promise((resolve) => rl.question(question, resolve));
}

// SIGINT handler
process.on('SIGINT', () => {
  console.log('\nWizard cancelled. No changes saved.');
  rl.close();
  process.exit(0);
});
```

**Wizard Section in core-config.yaml:**
```yaml
# Written/updated by `aiox wizard`
wizard:
  projectName: "my-project"
  template: "nextjs-react"
  agents:
    dev: true
    qa: true
    architect: true
    pm: true
    po: true
    sm: true
  completedAt: "2026-04-09T00:00:00.000Z"
```

**Config Merge Pattern (preserve all other keys):**
```javascript
const yaml = require('js-yaml');
const existing = yaml.load(fs.readFileSync('core-config.yaml', 'utf8')) || {};
existing.wizard = { ...answers, completedAt: new Date().toISOString() };
fs.writeFileSync('core-config.yaml', yaml.dump(existing));
```

**Re-run Detection:**
Read `config.wizard` before starting. Pass each field as `currentValue` to the respective step. Steps display `[Enter to keep: {currentValue}]` and return `currentValue` when user presses Enter on empty input.

**Testing:**
- Test location: `tests/wizard/` following existing test directory convention
- Mock readline using Jest's manual mock or `jest.spyOn(readline, 'createInterface')`
- All step tests should be pure functions testable without TTY

- `core-config.yaml` structure: wizard writes only under `wizard:` key, never touches other top-level keys
