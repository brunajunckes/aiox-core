# Story 6.2: Story Progress Tracker & Burndown

## Objective

Implement `aiox progress` — a CLI command that reads all story files from `docs/stories/`, groups them by sprint, and displays a visual burndown with per-status counts, completion percentage, and an ASCII chart showing how story counts evolved over time. A `--json` flag enables machine-readable output for integration with other tools.

## Agent

@dev (Dex)

## Status

Draft

## Story

**As a** developer or Scrum Master tracking sprint progress,
**I want** a single `aiox progress` command that shows burndown data from story files,
**so that** I can see sprint health at a glance without opening individual story documents.

## Acceptance Criteria

- [ ] `aiox progress` reads all `*.story.md` files from `docs/stories/` recursively
- [ ] Stories are grouped by sprint using the leading number in the filename (e.g. `6.1.story.md` → Sprint 6)
- [ ] Output shows per-sprint rows with columns: Sprint, Total, Done, In Progress, Draft, Completion %
- [ ] Completion % is calculated as `Done / Total * 100`, rounded to the nearest integer
- [ ] `aiox progress --sprint 5` filters output to only Sprint 5 stories and shows individual story titles + statuses
- [ ] An ASCII bar chart is rendered below the table showing completion % per sprint (newest sprint at bottom)
- [ ] `aiox progress --json` outputs the same data as minified JSON to stdout with no decorative text
- [ ] Status is read from the `## Status` section of each story file (values: Draft, Ready, InProgress, InReview, Done)
- [ ] Missing or malformed status fields are counted as Draft with a warning printed to stderr
- [ ] Unit tests cover: file parsing, sprint grouping, percentage calculation, JSON output format, `--sprint` filter

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
`aiox progress` is the canonical burndown view. No UI required. Output must be useful when piped: `aiox progress --json | jq '.sprints[-1].pct'`.

**Zero External Dependencies:**
Use only Node.js built-ins: `fs`, `path`, `process`. No chart libraries.

**Status Extraction Regex:**
```javascript
const STATUS_RE = /^##\s+Status\s*\n+([^\n]+)/m;
const match = content.match(STATUS_RE);
const status = match ? match[1].trim() : null;
```

**Filename Sprint Extraction:**
```javascript
// "6.1.story.md" → sprint 6
const sprintNum = parseInt(path.basename(file).split('.')[0], 10);
```

**JSON Output Shape (stable contract):**
```json
{
  "sprints": [
    {
      "sprint": 6,
      "total": 4,
      "done": 1,
      "inProgress": 1,
      "inReview": 0,
      "ready": 0,
      "draft": 2,
      "pct": 25,
      "stories": [
        { "id": "6.1", "title": "CLI Auto-Complete & Shell Integration", "status": "Draft" }
      ]
    }
  ]
}
```

**Testing:**
- Test location: `tests/unit/progress/`
- Use in-memory fixture strings instead of real file I/O in unit tests
- Snapshot-test the ASCII table and chart output against known inputs

**Relevant Source Tree:**
- Story files: `docs/stories/*.story.md`
- CLI router: `bin/aiox.js`
- Existing command modules: `packages/cli/commands/`
