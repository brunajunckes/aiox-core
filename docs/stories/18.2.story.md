# Story 18.2: Test Flaky Detector

## Objective

Detect intermittently failing test suites from historical test run data.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer,
**I want** to identify flaky tests from historical runs,
**so that** I can prioritize fixing unreliable tests.

## Acceptance Criteria

- [x] `aiox flaky` shows test suites that have failed intermittently from `.aiox/test-runs.jsonl`
- [x] `aiox flaky --threshold 2` shows tests that failed N+ times
- [x] `aiox flaky --record` records current test run results to history
- [x] `aiox flaky --clear` clears history
- [x] `aiox flaky --top N` shows top N most flaky tests
- [x] Command registered in bin/aiox.js
- [x] 15+ tests passing

## Scope

**IN:** flaky detection, history recording, threshold filtering, top N
**OUT:** Automatic test rerun, CI integration
