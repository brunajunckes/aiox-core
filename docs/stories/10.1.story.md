# Story 10.1: Version Bump & Release Notes Generator

## Objective

Provide an `aiox release-notes` command that generates categorized release notes from git history using conventional commits, supporting markdown and JSON output formats.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer releasing a new version of AIOX,
**I want** to generate release notes from git history automatically,
**so that** I have consistent, categorized changelogs without manual effort.

## Acceptance Criteria

- [x] `aiox release-notes` generates release notes from git log since last tag
- [x] `aiox release-notes --format markdown` outputs as markdown (default)
- [x] `aiox release-notes --format json` outputs as JSON
- [x] `aiox release-notes --since <tag>` uses a custom starting point
- [x] Parses conventional commits (feat:, fix:, docs:, etc.) into categorized sections
- [x] Handles missing tags gracefully (shows all commits)
- [x] Breaking changes are marked with **BREAKING**
- [x] Scoped commits display the scope in bold
- [x] Non-conventional commits appear in an "Other" section
- [x] Command registered in bin/aiox.js

## Tasks

- [x] Create command module at `.aiox-core/cli/commands/release-notes/index.js`
- [x] Implement git tag detection and log parsing
- [x] Implement conventional commit parser
- [x] Implement markdown and JSON formatters
- [x] Register command in `bin/aiox.js`
- [x] Write tests with 29 test cases

## Dev Notes

- Uses `child_process.execSync` for git operations with injectable exec function for testing
- All git operations have graceful error handling (returns null/empty on failure)
- Conventional commit regex: `/^(\w+)(?:\(([^)]*)\))?(!?):\s*(.*)$/`
