# Story 16.4: Feature Flags

## Objective

Implement a feature flag management system via `aiox feature-flags` that supports boolean and typed values with environment variable overrides and atomic file persistence.

## Agent

@dev (Dex)

## Status

InProgress

## Story

**As a** developer using the AIOX CLI,
**I want** to manage feature flags with environment variable overrides,
**so that** I can control features across environments without code changes.

## Acceptance Criteria

- [x] Feature flags stored in `.aiox/feature-flags.json`
- [x] createFeatureFlags(options) factory with configurable defaults
- [x] setFlag(name, value) sets boolean, string, or number flags
- [x] getFlag(name, defaultValue) retrieves flag with fallback
- [x] isEnabled(name) checks if boolean flag is true
- [x] listFlags() returns all flags including defaults
- [x] deleteFlag(name) removes a flag
- [x] toggleFlag(name) flips boolean value and returns new value
- [x] getOverrides() returns all AIOX_FLAG_* environment variable overrides
- [x] Environment variable format: AIOX_FLAG_<UPPERCASED_NAME> (hyphens to underscores)
- [x] Environment variables take precedence over file > defaults
- [x] Atomic writes with .tmp file + rename pattern
- [x] JSON file format with pretty-printing (2-space indent)
- [x] Default values support in factory options
- [x] `aiox feature-flags set <name> <value>` sets flag
- [x] `aiox feature-flags get <name>` reads flag value (JSON output)
- [x] `aiox feature-flags list` shows all flags with env overrides highlighted
- [x] `aiox feature-flags list --json` JSON format
- [x] `aiox feature-flags toggle <name>` flips boolean flag
- [x] `aiox feature-flags delete <name>` removes flag
- [x] `aiox feature-flags delete --dry-run` previews deletion
- [x] 25+ tests in `.aiox-core/core/feature-flags/__tests__/feature-flags.test.js`
- [x] Command registered in `bin/aiox.js`
