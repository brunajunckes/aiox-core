# Story 25.1: Team Notes & Knowledge Base

## Objective

Provide a team-oriented notes system for storing and retrieving knowledge snippets, design decisions, meeting notes, and other project-related text directly from the CLI.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer working in a team,
**I want** to add, search, and manage notes from the CLI,
**so that** I can build a searchable knowledge base without leaving my terminal.

## Acceptance Criteria

- [x] `aiox notes add <title> --content "text"` adds a note with unique id, title, content, timestamps
- [x] `aiox notes list` lists all notes with id, title, and tags
- [x] `aiox notes get <id>` shows full note details
- [x] `aiox notes search <term>` searches notes by title and content (case-insensitive)
- [x] `aiox notes remove <id>` deletes a note by id
- [x] `aiox notes export` exports all notes as JSON to stdout
- [x] `aiox notes --tags "bug,feature"` filters notes by tags
- [x] Store: `.aiox/notes.json`
- [x] Zero external npm dependencies
- [x] 15+ unit tests covering all operations

## Scope

**IN:**
- Note CRUD operations (add, list, get, search, remove, export)
- Tag-based filtering
- JSON file storage in `.aiox/notes.json`
- Unit tests for all subcommands
- Registration in `bin/aiox.js`

**OUT:**
- Rich text / markdown rendering
- Collaborative editing
- Sync across machines
- GUI / dashboard
