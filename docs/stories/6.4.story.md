# Story 6.4: CLI Auto-Update Checker

## Objective

Implement a non-blocking background update checker that notifies the user when a newer version of the AIOX CLI is available. On CLI startup, if more than 24 hours have passed since the last check, a background async check fires against the npm registry. If a newer version is found, a single informational line is printed to stderr after the command completes. Users can force a manual check via `aiox update check` or permanently disable auto-checks via `aiox update disable`. Network failures are silent. Zero impact on startup time.

## Story

**As a** developer using AIOX,
**I want** to be notified when a newer version of the CLI is available,
**so that** I can keep my tools up to date without having to manually check npm or GitHub.

## Acceptance Criteria

- [ ] On every CLI startup, the update checker reads `.aiox/update-check.json` to determine when the last check was performed
- [ ] If the last check was more than 24 hours ago (or the file does not exist), an async background check is initiated without blocking or delaying the primary CLI command execution
- [ ] The version check queries the npm registry endpoint `https://registry.npmjs.org/aiox-core/latest` with a 2-second timeout
- [ ] If a newer version is available (semver comparison: latest > installed), a single notice is printed to stderr after the command completes: `Update available: {installed} → {latest}. Run: npm install -g aiox-core`
- [ ] If no newer version is available, nothing is printed — completely silent
- [ ] Network errors, timeouts, and JSON parse failures are caught and suppressed — they must not produce any output or affect exit codes
- [ ] After a successful check (regardless of whether a newer version was found), `.aiox/update-check.json` is written with `{ lastChecked: ISO8601, latestVersion: string, currentVersion: string }`
- [ ] `aiox update check` forces an immediate synchronous version check, prints the result (up-to-date or update notice), and updates `.aiox/update-check.json`
- [ ] `aiox update disable` writes `{ disabled: true }` to `.aiox/update-check.json`; subsequent startups skip the background check entirely
- [ ] `aiox update enable` re-enables auto-checks by removing the `disabled: true` flag from `.aiox/update-check.json`
- [ ] The check interval and registry URL are configurable via environment variables: `AIOX_UPDATE_INTERVAL_HOURS` (default `24`) and `AIOX_UPDATE_REGISTRY_URL` (default npm registry URL) — for testing and CI environments
- [ ] Unit tests cover: version comparison logic, 24-hour staleness check, disabled state skip, network error suppression, state file read/write, `aiox update check` output

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
Update checks are a CLI-only feature. The notice is printed to stderr. No UI, no dashboard notification. The primary command's stdout output is never contaminated.

**Zero External Dependencies:**
Use only Node.js built-ins: `https`, `fs`, `path`. No axios, node-fetch, or semver package. Semver comparison is implemented inline.

**Non-Blocking Pattern:**
```javascript
// In CLI init — fires AFTER primary command dispatch has started
function scheduleAutoCheck(currentVersion) {
  try {
    const state = readState();
    if (isDisabled(state) || !isStale(state)) return;

    // Runs async, does not block
    setImmediate(async () => {
      try {
        const latest = await fetchLatestVersion();
        if (!latest) return;
        writeState({ lastChecked: new Date().toISOString(), latestVersion: latest, currentVersion });
        if (isNewer(latest, currentVersion)) {
          process.on('exit', () => {
            process.stderr.write(
              `\nUpdate available: ${currentVersion} → ${latest}. Run: npm install -g aiox-core\n`
            );
          });
        }
      } catch {
        // Silently suppressed
      }
    });
  } catch {
    // Silently suppressed
  }
}
```

**HTTP Fetch with 2s Timeout:**
```javascript
const https = require('https');

function fetchLatestVersion() {
  return new Promise((resolve) => {
    const url = process.env.AIOX_UPDATE_REGISTRY_URL ||
      'https://registry.npmjs.org/aiox-core/latest';
    const req = https.get(url, (res) => {
      let data = '';
      res.on('data', (chunk) => (data += chunk));
      res.on('end', () => {
        try { resolve(JSON.parse(data).version || null); }
        catch { resolve(null); }
      });
    });
    req.setTimeout(2000, () => { req.destroy(); resolve(null); });
    req.on('error', () => resolve(null));
  });
}
```

**State File Shape:**
```json
{
  "lastChecked": "2026-04-09T12:00:00.000Z",
  "latestVersion": "2.3.1",
  "currentVersion": "2.3.0"
}
```

**Semver Comparison (no external deps):**
```javascript
function isNewer(latest, current) {
  const parse = (v) => v.replace(/-.*$/, '').split('.').map(Number);
  const [la, lb, lc] = parse(latest);
  const [ca, cb, cc] = parse(current);
  return la > ca || (la === ca && lb > cb) || (la === ca && lb === cb && lc > cc);
}
```
