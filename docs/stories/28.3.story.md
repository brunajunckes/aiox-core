# Story 28.3: Middleware Generator

## Objective

Generate middleware boilerplate with type-specific logic from a CLI command, providing ready-to-use patterns for common middleware concerns.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** to generate middleware files with type-specific boilerplate,
**so that** I can quickly scaffold auth, logging, validation, and CORS middleware.

## Acceptance Criteria

- [x] `aiox gen-middleware <name>` generates middleware boilerplate
- [x] `aiox gen-middleware <name> --type auth` generates auth-specific middleware
- [x] `aiox gen-middleware <name> --type logging` generates logging middleware
- [x] `aiox gen-middleware <name> --type validation` generates validation middleware
- [x] `aiox gen-middleware <name> --type cors` generates CORS middleware
- [x] `aiox gen-middleware --list` lists generated middlewares
- [x] Generates middleware.js + middleware.test.js
- [x] Shows help with no args or --help
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests passing

## Scope

**IN:**
- Middleware generation with type-specific logic
- Support for auth, logging, validation, cors, generic types
- Listing generated middlewares

**OUT:**
- Express/Koa/Fastify framework integration
- Middleware chaining configuration
- Runtime middleware loading
