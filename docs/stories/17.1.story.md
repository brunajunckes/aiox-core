# Story 17.1: Key-Value Store

## Objective

Provide a persistent key-value store accessible via CLI for storing arbitrary data used by agents, workflows, and user scripts.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** CLI user and agent system,
**I want** a simple key-value store with get/set/delete/list/clear/export/import operations,
**so that** I can persist and share configuration and runtime data across sessions.

## Acceptance Criteria

- [x] `aiox kv get <key>` returns the value for a key
- [x] `aiox kv set <key> <value>` stores a key-value pair
- [x] `aiox kv delete <key>` removes a key
- [x] `aiox kv list` lists all key-value pairs
- [x] `aiox kv clear` clears the entire store
- [x] `aiox kv export` outputs the store as JSON to stdout
- [x] `aiox kv import <file>` imports key-value pairs from a JSON file
- [x] Store persisted at `.aiox/kv-store.json`
- [x] Zero external npm dependencies
- [x] 15+ tests passing

## Scope

**IN:**
- CRUD operations on key-value pairs
- JSON persistence
- Export/import functionality
- CLI runner with help text

**OUT:**
- Encryption
- Remote sync
- TTL/expiration (handled by cache command)
