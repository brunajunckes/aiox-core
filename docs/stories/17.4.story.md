# Story 17.4: Backup & Restore

## Objective

Provide full directory-level backup and restore of the `.aiox/` directory using tar.gz archives, with listing, dry-run restore, and cleanup of old backups.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** CLI user,
**I want** to create timestamped backups of my entire .aiox/ directory and restore them,
**so that** I can recover from configuration corruption or experiment safely.

## Acceptance Criteria

- [x] `aiox backup create` creates timestamped backup of .aiox/ as tar.gz
- [x] `aiox backup list` lists available backups from `.aiox/backups/`
- [x] `aiox backup restore <name>` restores from backup
- [x] `aiox backup restore <name> --dry-run` previews restore
- [x] `aiox backup clean --keep 5` keeps only N most recent backups
- [x] Zero external npm dependencies (uses Node.js child_process + tar)
- [x] 15+ tests passing

## Scope

**IN:**
- tar.gz creation/extraction via system tar
- Backup listing and cleanup
- Dry-run restore preview
- Exclude backups/ from archive

**OUT:**
- Compression level options
- Remote backup storage
- Incremental backups
- Encryption
