# Story 11.2: Plugin Loader & Registry

## Objective

Provide a CLI command to manage plugins in `.aiox/plugins/` with list, info, enable, disable, and init subcommands.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer extending the AIOX CLI,
**I want** a plugin system that discovers, loads, and manages plugins,
**so that** I can extend AIOX functionality through a modular plugin architecture.

## Acceptance Criteria

- [x] `aiox plugins list` lists installed plugins from `.aiox/plugins/`
- [x] `aiox plugins info <name>` shows plugin manifest details
- [x] `aiox plugins enable <name>` enables a plugin
- [x] `aiox plugins disable <name>` disables a plugin
- [x] `aiox plugins init <name>` scaffolds a new plugin directory
- [x] Plugin manifest: `plugin.json` with { name, version, description, commands, hooks, enabled }
- [x] `loadPlugins()` discovers and loads enabled plugins
- [x] Command registered in `bin/aiox.js`
- [x] 15+ tests in `tests/cli/plugins.test.js`
