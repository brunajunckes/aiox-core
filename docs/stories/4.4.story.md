# Story 4.4: Squads Marketplace MVP

## Objective

Build the MVP of the Squads Marketplace — a CLI-first distribution system that allows users to publish, search, and install AIOX squads using GitHub Releases as the backend. Starter registry ships with 2-3 example squads. No web UI required.

## Agent

@dev (Dex) + @devops (Gage)

## Status

Draft

## Story

**As a** developer using AIOX,
**I want** to discover, publish, and install community-built squads via the CLI,
**so that** I can extend AIOX with pre-built agent configurations without manual setup.

## Acceptance Criteria

- [ ] `aiox squads publish` packages the current squad directory as a `.tar.gz` tarball and creates a GitHub Release in the user's specified repo
- [ ] `aiox squads search <query>` searches published squads and displays results (name, description, author, version, install count)
- [ ] `aiox squads install <name>` downloads and installs a squad from the registry into `squads/`
- [ ] Starter registry includes at minimum 2 example squads: `claude-code-mastery` (already exists) and one additional squad
- [ ] GitHub Releases is used as the distribution backend — tarballs are attached as release assets
- [ ] `aiox squads list` displays currently installed squads with name, version, and source
- [ ] Squad package format: `aiox-squad-{name}-{version}.tar.gz` containing `squad.yaml` manifest + squad files
- [ ] `squad.yaml` manifest schema: `{ name, version, description, author, aiox_version_min, files[] }`
- [ ] `aiox squads install` validates the `squad.yaml` manifest before installing (version compatibility check against running AIOX version)
- [ ] All commands are CLI-only — no web UI, no browser required
- [ ] Unit tests cover: pack/unpack squad tarball, manifest validation, version compatibility check

## Dev Notes

**GitHub Releases as Backend:**
The publish command uses `gh release create` to push tarballs. The search/install commands use the GitHub API (`gh api /repos/{owner}/{repo}/releases`) to fetch release metadata. The registry is a JSON file hosted in a known AIOX GitHub repo (define as constant):
```javascript
const REGISTRY_REPO = 'synkra/aiox-squads'; // where registry JSON lives
const REGISTRY_URL = 'https://raw.githubusercontent.com/synkra/aiox-squads/main/registry.json';
```

**Squad Install Directory:**
Install to `squads/{name}/` relative to the project root (consistent with `squads/` in the existing project structure per CLAUDE.md).

**Path Traversal Guard:**
```javascript
function safePath(base, userPath) {
  const resolved = path.resolve(base, userPath);
  if (!resolved.startsWith(path.resolve(base))) {
    throw new Error(`Path traversal detected: ${userPath}`);
  }
  return resolved;
}
```

**Installed Squads Registry:**
`.aiox/squads/installed.json` schema:
```json
{
  "installed": [
    {
      "name": "claude-code-mastery",
      "version": "1.0.0",
      "source": "registry",
      "installedAt": "2026-04-08T12:00:00Z"
    }
  ]
}
```

**`squad.yaml` Schema:**
```yaml
name: claude-code-mastery
version: 1.0.0
description: "Optimized squad for Claude Code workflows"
author: synkra
aiox_version_min: 2.1.0
files:
  - agents/
  - workflows/
  - README.md
```

**Version Compatibility:**
Use semver comparison. `aiox_version_min` means the installed AIOX version must be >= this value. Get current AIOX version from `package.json` in the AIOX install directory or from `aiox --version` output.

**Network Calls:**
- Use `https` built-in module (no `node-fetch` or `axios`)
- Set `timeout: 5000` on all requests
- On timeout or network error: use cached registry if available, else print clear error

**File Locations:**
- Registry cache: `.aiox/squads/registry-cache.json`
- Install records: `.aiox/squads/installed.json`
- Downloaded tarballs (temp): `.aiox/squads/downloads/` (cleaned after install)
- Module: consistent with other CLI modules from Stories 4.1-4.3
- Tests: `tests/squads/`

**Testing:**
- Mock `https.get` for registry and download tests
- Test path traversal with entries like `../../../etc/passwd`
- Mock `gh` CLI calls using child_process mocking
- Do not make real network calls in unit tests

## Dependencies

- `gh` CLI (required for `aiox squads publish`, optional for search/install if registry cache is warm)
- Stories 4.1-4.3 are independent from this story (no functional dependency)
