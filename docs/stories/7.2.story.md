# Story 7.2: Dependency Audit & Security Scanner

## Objective

Add a security and dependency health command to the AIOX CLI that wraps `npm audit` with additional custom checks for outdated packages, unused dependencies, and license compliance. The command produces actionable output in the terminal and can generate a markdown report. No external npm packages are added — the command wraps Node.js built-ins and the npm CLI that is already present in any Node.js environment.

## Agent

@dev (Dex)

## Status

Draft

## Story

**As a** developer maintaining an AIOX-managed project,
**I want** a single command that audits dependencies for security vulnerabilities, outdated packages, unused deps, and license issues,
**so that** I can identify and fix security risks without context-switching to multiple tools or external services.

## Acceptance Criteria

- [ ] `aiox audit` runs a full dependency audit and prints a summary to stdout
- [ ] `aiox audit --fix` auto-fixes vulnerabilities where `npm audit fix` supports it
- [ ] `aiox audit report` generates a markdown security report at `docs/audit-report-<YYYY-MM-DD>.md`
- [ ] Audit checks cover: known CVEs (via `npm audit --json`), outdated packages (via `npm outdated --json`), unused dependencies (static import scan of source files vs `package.json` dependencies), and license compliance (reads `license` field from each installed package's `package.json`)
- [ ] License compliance check flags packages with licenses not in an allowlist (default allowlist: MIT, ISC, Apache-2.0, BSD-2-Clause, BSD-3-Clause, 0BSD, CC0-1.0)
- [ ] Unused dependency detection scans `packages/` and `bin/` source files for `require()`/`import` statements and compares against `dependencies` in root `package.json`
- [ ] Exit code is 0 when no HIGH/CRITICAL vulnerabilities found; exit code 1 when any HIGH/CRITICAL found
- [ ] `aiox audit --json` outputs machine-readable JSON of all findings
- [ ] Zero external npm dependencies — wraps `npm audit`, `npm outdated`, and Node.js `fs`/`child_process` built-ins only
- [ ] Unit tests cover: CVE summary parsing, outdated package parsing, unused dep detection logic, license flag logic

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
All audit logic is CLI-only. No UI surface required. Reports written to `docs/` (L4 — always mutable).

**npm CLI Availability:**
`npm` is guaranteed to be on PATH in any Node.js development environment. Use `child_process.execSync('npm audit --json', { encoding: 'utf8' })`. Catch non-zero exits (npm audit exits 1 when vulns found) and parse stdout regardless — npm still emits JSON on non-zero exit.

**npm audit JSON Shape (npm v8+):**
```json
{
  "auditReportVersion": 2,
  "vulnerabilities": { "<pkg>": { "severity": "high", ... } },
  "metadata": { "vulnerabilities": { "critical": 0, "high": 2, ... } }
}
```
Parse `metadata.vulnerabilities` for the summary counts.

**npm outdated JSON Shape:**
```json
{ "<pkg>": { "current": "1.2.3", "wanted": "1.2.4", "latest": "2.0.0" } }
```
npm outdated exits 1 when any package is outdated — catch and parse stdout anyway.

**Static Import Scanner Regex:**
```js
// require
/require\(['"]([^'"@.][^'"]*)['"]\)/g
// ESM import
/from\s+['"]([^'"@.][^'"]*)['"]/g
```
Strip subpath imports: `lodash/merge` → `lodash`. Ignore relative paths and scoped-package subpaths carefully.

**Relevant Source Tree:**
- CLI router: `bin/aiox.js`
- Existing command modules: `packages/cli/commands/`
- Root package.json: `/root/package.json`
- Source to scan: `packages/`, `bin/`

**Testing:**

- Test location: `tests/unit/audit/`
- Mock all `child_process.execSync` calls — do not invoke real npm in unit tests
- Use fixture JSON strings matching the npm audit and npm outdated schemas
- Fixture `node_modules` for license checker: create temp directory in `beforeAll`, clean in `afterAll`
