# Story 26.1: Web Dashboard Server

## Objective

Provide a built-in web dashboard via `aiox web` that serves a single-page HTML dashboard and JSON API endpoints for project stats, stories, commands, and health status.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using AIOX,
**I want** a web dashboard accessible via browser,
**so that** I can visualize project stats, story progress, and command inventory in a rich HTML interface.

## Acceptance Criteria

- [x] `aiox web` starts HTTP server on port 4000 serving a single-page dashboard
- [x] `aiox web --port 4567` allows custom port
- [x] Serves static HTML from a template string (no external files needed)
- [x] Dashboard HTML includes: project stats, story list, command count, test status
- [x] Fetches data from internal functions (same as aiox stats-summary, sprint-report, etc.)
- [x] Uses Node.js http.createServer + inline HTML/CSS/JS
- [x] `aiox web --api-only` only serves JSON API endpoints, no HTML
- [x] GET / serves HTML dashboard
- [x] GET /api/dashboard returns full dashboard data as JSON
- [x] GET /api/stories returns story list as JSON
- [x] GET /api/commands returns command list as JSON
- [x] GET /api/health returns health check as JSON
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests in tests/cli/web.test.js

## Scope

**IN:**
- HTTP server with HTML dashboard
- JSON API endpoints
- Data collection from project files
- API-only mode flag

**OUT:**
- WebSocket live updates
- Authentication
- HTTPS/TLS
