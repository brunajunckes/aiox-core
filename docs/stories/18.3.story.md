# Story 18.3: Test Impact Analysis

## Objective

Analyze which test files cover which source files, and suggest tests to run for changed files.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer,
**I want** to know which tests are affected by my code changes,
**so that** I can run only the relevant tests and save time.

## Acceptance Criteria

- [x] `aiox test-impact` analyzes which test files cover which source files
- [x] `aiox test-impact <file>` shows which tests would be affected by changing a file
- [x] `aiox test-impact --format json` outputs as JSON
- [x] `aiox test-impact --suggest` suggests which tests to run for uncommitted changes
- [x] Maps source to test by naming convention (foo.js -> foo.test.js) and import analysis
- [x] Command registered in bin/aiox.js
- [x] 15+ tests passing

## Scope

**IN:** source-to-test mapping, import parsing, git diff integration, JSON output
**OUT:** Coverage-based mapping, watch mode
