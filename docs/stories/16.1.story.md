# Story 16.1: Local REST API Server

## Objective

Implement a local REST API server via `aiox serve` that exposes project status, stories, commands, and health endpoints over HTTP using Node.js built-in `http` module.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** a local REST API server,
**so that** I can query project status, stories, and commands programmatically.

## Acceptance Criteria

- [x] `aiox serve` starts HTTP server on port 3456 (configurable with --port)
- [x] `aiox serve --port 8080` starts on custom port
- [x] GET /api/status returns project status JSON
- [x] GET /api/stories returns list of stories
- [x] GET /api/commands returns registered CLI commands
- [x] GET /api/health returns health check
- [x] Returns JSON responses with correct Content-Type
- [x] Graceful shutdown on SIGINT/SIGTERM
- [x] Uses Node.js `http.createServer` (zero deps)
- [x] 15+ tests in `tests/cli/serve.test.js`
- [x] Command registered in `bin/aiox.js`

## Tasks / Subtasks

- [x] Task 1: Implement route handler with status/stories/commands/health endpoints
- [x] Task 2: Implement port parsing and validation
- [x] Task 3: Implement server creation with graceful shutdown
- [x] Task 4: Register command in bin/aiox.js
- [x] Task 5: Write 15+ tests covering handler functions
