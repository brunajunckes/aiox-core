# Story 15.2: Watch Mode for File Changes

## Objective

Implement a file watcher that monitors project files and runs commands on change, using `fs.watch` with debouncing and configurable patterns.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** a watch mode that re-runs commands when files change,
**so that** I get instant feedback during development.

## Acceptance Criteria

- [x] `aiox watch` watches src files, runs tests on change
- [x] `aiox watch --command "npm run lint"` custom command on file change
- [x] `aiox watch --pattern "**/*.js"` custom glob pattern
- [x] `aiox watch --debounce 500` debounce interval in ms (default: 300)
- [x] Uses `fs.watch` recursively on project dirs
- [x] Ignores node_modules, .git, coverage, dist
- [x] 15+ tests in `tests/cli/watch.test.js`
- [x] Command registered in `bin/aiox.js`

## Tasks / Subtasks

- [x] Task 1: Implement argument parser with defaults
- [x] Task 2: Implement glob pattern matching for file paths
- [x] Task 3: Implement directory discovery with ignore list
- [x] Task 4: Implement debounce utility
- [x] Task 5: Implement watch start/stop with event callbacks
- [x] Task 6: Register command in bin/aiox.js
- [x] Task 7: Write 15+ tests covering all functions
