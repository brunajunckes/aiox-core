# Story 24.3: Error Rate Monitor

## Objective

Track and display error rates from a JSONL log with filtering and top-N analysis.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer,
**I want** an error rate monitor,
**so that** I can track error frequency and identify recurring issues.

## Acceptance Criteria

- [x] `aiox error-rate` shows error rates from `.aiox/error-log.jsonl`
- [x] `aiox error-rate --since 24h` filters by time
- [x] `aiox error-rate --top 5` shows top N most frequent errors
- [x] `aiox error-rate --format json` outputs as JSON
- [x] `aiox error-rate --record "<error-message>"` logs an error
- [x] `aiox error-rate --clear` clears log
- [x] Command registered in bin/aiox.js
- [x] 15+ tests passing
