# Story 5.1: CLI Smoke Test Suite

## Objective

Implement an automated smoke test runner that validates all AIOX CLI commands work end-to-end after each release or feature addition. Running `aiox smoke-test` executes a curated set of command invocations, reports pass/fail per command, and exits with code 0 (all pass) or 1 (any fail) — enabling regression detection as a single CLI command.

## Agent

@dev (Dex)

## Status

Draft

## Story

**As a** developer maintaining or extending AIOX,
**I want** a single `aiox smoke-test` command that validates every CLI command works end-to-end,
**so that** I can catch regressions immediately after merging new features without running manual checks.

## Acceptance Criteria

- [ ] `aiox smoke-test` runs a suite of invocations covering: agents, palette, status, resume, help, telemetry, experiment, feedback, squads
- [ ] Each command invocation is reported individually as PASS or FAIL with the command that was run
- [ ] Suite exits with code 0 if all commands pass, code 1 if any command fails
- [ ] Failed commands include the stderr/exit code captured for diagnosis
- [ ] `aiox smoke-test --verbose` prints full stdout/stderr for every command (not just failures)
- [ ] `aiox smoke-test --bail` stops execution at the first failure
- [ ] Smoke test does not mutate permanent state (uses dry-run flags or temp directories where commands would write files)
- [ ] Output includes a final summary line: `Smoke Test: X/Y passed`
- [ ] Unit tests cover: runner logic, pass/fail aggregation, exit code behavior, bail flag

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
The smoke test suite IS the CLI quality gate. No UI, no dashboard. `aiox smoke-test` is the canonical validation path.

**Zero External Dependencies:**
Use only Node.js built-ins: `child_process`, `process`, `os`, `path`. No new npm packages.

**State Isolation Pattern:**
Commands that write to `.aiox/` should be invoked with a temporary `AIOX_HOME` env override or use existing `--dry-run` flags. For commands without dry-run, capture output and verify exit code only — do not assert on file side effects.

**Command Manifest Shape:**
```javascript
const SMOKE_COMMANDS = [
  { name: 'help',      args: ['help'] },
  { name: 'status',    args: ['status'] },
  { name: 'agents',    args: ['agents', '--list'] },
  { name: 'palette',   args: ['palette', '--list'] },
  { name: 'telemetry', args: ['telemetry', 'status'] },
  { name: 'squads',    args: ['squads', '--list'] },
  // ... others
];
```

**Subprocess Pattern:**
```javascript
const result = spawnSync(process.execPath, [cliBin, ...args], {
  encoding: 'utf8',
  env: { ...process.env, AIOX_HOME: tmpDir },
});
const passed = result.status === (cmd.expectExitCode ?? 0);
```

**Testing:**
- Test location: `tests/smoke/` or matching existing test directory convention
- Mock `spawnSync` in unit tests — do not actually invoke CLI subprocesses in unit tests
