# Story 27.2: Risk Register

## Objective

Provide a CLI command `aiox risks` to maintain a risk register with severity levels, mitigation plans, and status tracking.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** project manager using AIOX,
**I want** to register and track project risks with severity and mitigation plans,
**so that** I can proactively manage project threats.

## Acceptance Criteria

- [x] `aiox risks add "<title>" --severity high --mitigation "<plan>"` adds a risk
- [x] `aiox risks list` lists all risks sorted by severity
- [x] `aiox risks update <id> --status mitigated` updates risk status
- [x] `aiox risks remove <id>` deletes a risk
- [x] `aiox risks --format json` outputs as JSON
- [x] Severity levels: critical, high, medium, low
- [x] Status values: open, mitigated, closed
- [x] Store: `.aiox/risks.json`
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests in tests/cli/risks.test.js

## Scope

**IN:** Add, list, update, remove risks; severity sorting; JSON output
**OUT:** Risk scoring matrix, probability calculation, risk heat maps
