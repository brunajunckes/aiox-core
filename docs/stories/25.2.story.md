# Story 25.2: Decision Log

## Objective

Provide a structured decision log so teams can record, search, and review architectural and technical decisions made during development.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** team lead or architect,
**I want** to log decisions with context, outcome, and alternatives,
**so that** the team can understand why choices were made and revisit them later.

## Acceptance Criteria

- [x] `aiox decisions add "<title>" --context "<why>" --outcome "<what>"` logs a decision with unique id, title, context, outcome, alternatives, date, author
- [x] `aiox decisions list` lists all decisions with id, date, and title
- [x] `aiox decisions get <id>` shows full decision details
- [x] `aiox decisions search <term>` searches by title, context, and outcome
- [x] `aiox decisions remove <id>` deletes a decision
- [x] `aiox decisions --format json` outputs as JSON
- [x] Fields: id, title, context, outcome, alternatives, date, author
- [x] Store: `.aiox/decisions.json`
- [x] Zero external npm dependencies
- [x] 15+ unit tests covering all operations

## Scope

**IN:**
- Decision CRUD operations
- Search across title, context, outcome
- JSON and table output formats
- Unit tests for all subcommands
- Registration in `bin/aiox.js`

**OUT:**
- Decision approval workflows
- Integration with external ADR tools
- Sync / collaboration features
