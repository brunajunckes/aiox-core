# Story 28.4: Config Schema Generator

## Objective

Generate configuration modules with typed defaults, validation, and optional environment variable support from a CLI command.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** to generate config modules with validation and defaults,
**so that** I can quickly scaffold type-safe configuration with optional env var support.

## Acceptance Criteria

- [x] `aiox gen-config <name>` generates config module with validation
- [x] `aiox gen-config <name> --fields "port:number:3000,host:string:localhost"` with defaults
- [x] `aiox gen-config <name> --env` generates environment-variable-aware config
- [x] `aiox gen-config --list` lists generated configs
- [x] Generates config.js with getDefaults() + validate() + load() + config.test.js
- [x] Shows help with no args or --help
- [x] Validates config name is provided
- [x] Supports types: string, number, boolean
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests passing

## Scope

**IN:**
- Config generation with typed fields and defaults
- Validation function generation
- Environment variable support (--env)
- Listing generated configs

**OUT:**
- YAML/TOML config file parsing
- Config hot-reloading
- Secret management integration
