# Story 25.4: Review Request Manager

## Objective

Provide a lightweight code review request system that tracks review requests, their status, verdicts, and aggregate metrics.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer wanting code review,
**I want** to create review requests with file lists and track their completion,
**so that** the team has visibility into review workload and turnaround times.

## Acceptance Criteria

- [x] `aiox reviews create "<title>" --files "a.js,b.js"` creates a review request with unique id, title, files, status, timestamps
- [x] `aiox reviews list` lists all reviews with id, status, and title
- [x] `aiox reviews complete <id> --verdict approve|reject` completes a review with verdict
- [x] `aiox reviews stats` shows review metrics: total, pending, completed, approved, rejected, completion rate, avg time
- [x] `aiox reviews --format json` outputs as JSON
- [x] Store: `.aiox/reviews.json`
- [x] Zero external npm dependencies
- [x] 15+ unit tests covering all operations

## Scope

**IN:**
- Review request CRUD (create, list, complete)
- Stats/metrics calculation
- JSON and table output formats
- Unit tests for all subcommands
- Registration in `bin/aiox.js`

**OUT:**
- GitHub PR integration
- Reviewer assignment / rotation
- Notification system
- Dashboard visualization
