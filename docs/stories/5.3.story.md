# Story 5.3: Project Health Dashboard CLI

## Objective

Implement `aiox health` to display a comprehensive project health report in the terminal. Aggregates data from existing AIOX modules — test status, lint status, story progress, telemetry summary, and installed squads — into a single, glanceable report. Supports `--json` for machine-readable output suitable for CI pipelines.

## Status

## Story

**As a** developer or team lead working on an AIOX-managed project,
**I want** `aiox health` to give me a single-command overview of the project's current state,
**so that** I can quickly identify which areas need attention without running multiple separate commands.

## Acceptance Criteria

- [ ] `aiox health` prints a health report with the following sections: Test Status, Lint Status, Story Progress, Telemetry Summary, Installed Squads
- [ ] Test Status: shows last test run result (pass/fail/unknown) and count of passing vs failing tests if available
- [ ] Lint Status: shows last lint run result (clean/errors/unknown)
- [ ] Story Progress: shows count of stories by status (Draft, Ready, InProgress, InReview, Done) from `docs/stories/`
- [ ] Telemetry Summary: shows telemetry enabled/disabled and total events recorded (from `.aiox/metrics/`)
- [ ] Installed Squads: lists squad names and versions from `.aiox/squads/` or equivalent registry
- [ ] Each section has a clear health indicator: OK, WARN, or ERROR
- [ ] `aiox health --json` outputs all sections as structured JSON with the same data
- [ ] Sections with unavailable data (e.g., no test results cached) show status UNKNOWN — never crash
- [ ] Unit tests cover: story count parsing, section aggregation, JSON output schema, UNKNOWN fallback behavior

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
`aiox health` is purely observational — it reads and reports. No state modification. No UI required.

**Zero External Dependencies:**
Use Node.js built-ins: `fs`, `path`, `os`. No new npm packages.

**Graceful Degradation Pattern:**
```javascript
function safeRead(fn, fallback) {
  try {
    return fn();
  } catch {
    return fallback;
  }
}

const telemetry = safeRead(() => readTelemetryState(), { status: 'UNKNOWN' });
```

**Story Status Parsing Pattern:**
```javascript
function parseStoryStatus(content) {
  const m = content.match(/^## Status\s*\n+(.+)/m);
  return m ? m[1].trim() : 'Unknown';
}
```

**Health Indicator Logic:**
- Test: UNKNOWN if no cache, OK if last run passed, ERROR if last run failed
- Lint: UNKNOWN if no cache, OK if clean, WARN if warnings, ERROR if errors
- Stories: OK if 0 stories InProgress/InReview for > 14 days (future: basic, today just count)
- Telemetry: OK (shows state), no health judgment
- Squads: OK (lists installed), no health judgment

**JSON Output Shape:**
```json
{
  "generatedAt": "2026-04-08T12:00:00Z",
  "sections": {
    "testStatus": { "indicator": "OK", "passing": 42, "failing": 0 },
    "lintStatus": { "indicator": "UNKNOWN" },
    "storyProgress": { "indicator": "OK", "Draft": 2, "InProgress": 1, "Done": 13 },
    "telemetry": { "indicator": "OK", "enabled": false, "totalEvents": 0 },
    "squads": { "indicator": "OK", "installed": [] }
  }
}
```

**Testing:**
- Mock `fs.readFileSync` and `fs.readdirSync` in unit tests
- Test location: `tests/health/` or matching existing test directory convention
