# Story 26.3: Status Page Generator

## Objective

Provide a status page generator via `aiox status-page` that produces a self-contained HTML page showing system service health, test status, and system information.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using AIOX,
**I want** to generate a status page showing system health,
**so that** I can quickly see which services are operational and share the status with my team.

## Acceptance Criteria

- [x] `aiox status-page` generates HTML status page to stdout
- [x] `aiox status-page --output status.html` writes to file
- [x] `aiox status-page --json` outputs data as JSON
- [x] Includes service status: Node.js, Git, npm, Ollama
- [x] Includes last test run timestamp
- [x] Includes system uptime and platform info
- [x] HTML is self-contained (inline CSS, no external deps)
- [x] Shows "All Systems Operational" or "Degraded" overall status
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests in tests/cli/status-page.test.js

## Scope

**IN:**
- Service health checks (Node, Git, npm, Ollama)
- HTML generation with inline CSS
- JSON output mode
- File output option

**OUT:**
- Real-time monitoring
- Historical status tracking
- Email/webhook notifications
