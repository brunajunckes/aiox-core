# Story 16.2: Webhook Handler

## Objective

Implement a webhook management system via `aiox webhooks` that allows registering, removing, testing, and firing webhooks for AIOX system events.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using the AIOX CLI,
**I want** to manage webhooks for system events,
**so that** I can integrate AIOX with external services via HTTP callbacks.

## Acceptance Criteria

- [x] `aiox webhooks list` lists registered webhooks from `.aiox/webhooks.yaml`
- [x] `aiox webhooks add <url> --event <event>` registers webhook for event
- [x] `aiox webhooks remove <id>` removes webhook
- [x] `aiox webhooks test <id>` sends test payload to webhook URL
- [x] `aiox webhooks fire <event> --data '{"key":"val"}'` fires event to all registered hooks
- [x] Events: story.completed, test.passed, test.failed, deploy.started
- [x] Webhooks persisted in YAML-like format
- [x] 15+ tests in `tests/cli/webhooks.test.js`
- [x] Command registered in `bin/aiox.js`

## Tasks / Subtasks

- [x] Task 1: Implement YAML-like parser/serializer for webhooks file
- [x] Task 2: Implement add/remove/get/list webhook functions
- [x] Task 3: Implement HTTP payload sender with timeout
- [x] Task 4: Implement test and fire subcommands
- [x] Task 5: Register command in bin/aiox.js
- [x] Task 6: Write 15+ tests covering all functions
