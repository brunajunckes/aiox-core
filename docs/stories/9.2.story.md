Implement a coverage threshold enforcement system that tracks test coverage per module, reports coverage trends, and blocks PRs that drop coverage below configured thresholds. This ensures test quality doesn't regress as the codebase grows.

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
Coverage is a CLI-only observability tool. Reports stored in `.aiox/` (L4 — project runtime, always mutable). Config stored in `core-config.yaml` (L3 — project config, mutable with care).

**Jest Coverage Output:**
After `npm test -- --coverage`, Jest writes to `coverage/coverage-summary.json`:
```json
{
  "total": {
    "lines": { "total": 100, "covered": 95, "skipped": 0, "pct": 95.0 },
    "statements": { "total": 120, "covered": 114, "skipped": 0, "pct": 95.0 },
    "functions": { "total": 45, "covered": 42, "skipped": 0, "pct": 93.33 },
    "branches": { "total": 30, "covered": 27, "skipped": 0, "pct": 90.0 }
  },
  "packages/cli/commands/test.js": {
    "lines": { "pct": 92.0 },
    ...
  }
}
```

Do NOT re-run Jest. Parse the existing `coverage/coverage-summary.json` file directly.

**Module Name Extraction:**
From path `packages/cli/commands/test.js`, extract module as `packages.cli.commands.test` (replace `/` and `.js`). Provide option to configure module grouping (e.g., group all `packages/cli/*` as `packages.cli`).

**Threshold Config Structure:**
```yaml
quality:
  coverage:
    statements: 70
    branches: 60
    functions: 70
    lines: 70
    modules:
      packages.core: { statements: 85 }
      packages.cli: { statements: 75, branches: 70 }
```

**JSONL Format:**
Each line is a complete, valid JSON object. Use `\n` as delimiter (not `,`). Example:
```
{"timestamp":"2026-04-08T10:30:00Z","gitSha":"abc123","total":{"statements":95,"branches":90,"functions":93,"lines":95},"modules":[...]}
{"timestamp":"2026-04-08T10:45:00Z","gitSha":"def456","total":{"statements":95,"branches":90,"functions":93,"lines":95},"modules":[...]}
```

**Concurrent Write Safety:**
Use `fs.appendFileSync` for atomic append (no buffering). For concurrent reads during append, JSONL format guarantees that readers see complete lines only.

**CI Comment Format:**
Fetch baseline from `origin/main` (or `HEAD~1` if on main), parse its coverage report, compute deltas, format as markdown table:
```markdown
## Coverage Report

| Metric | Previous | Current | Delta |
|--------|----------|---------|-------|
| Statements | 94% | 95% | +1% ↑ |
| Branches | 88% | 90% | +2% ↑ |
| Functions | 92% | 93% | +1% ↑ |
| Lines | 94% | 95% | +1% ↑ |

✅ All thresholds met!
```

If baseline missing, comment: "Establishing baseline coverage for this branch."

**Graceful Fallback:**
If `coverage/coverage-summary.json` missing, message: "Coverage report not found. Run `npm test -- --coverage` to generate it."

If CI cannot fetch previous report, fallback to "first run" comment instead of failing the CI step.

**Relevant Source Tree:**
- CLI router: `bin/aiox.js`
- Jest coverage output: `coverage/coverage-summary.json`
- Coverage history: `.aiox/coverage-history.jsonl`
- Coverage report: `.aiox/coverage-report.json`
- Threshold config: `core-config.yaml` → `quality.coverage`
- CI workflow: `.github/workflows/` (existing test workflow)

**Testing:**

- Test location: `tests/unit/coverage/`
- Mock all `fs` operations except for pure function tests
- Test fixtures: `tests/fixtures/coverage/coverage-summary.json` (sample Jest output)
- Use `jest.spyOn(fs, 'appendFileSync')` for history recorder tests
