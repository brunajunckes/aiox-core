# Story 26.4: Markdown Report Engine

## Objective

Provide a markdown report engine via `aiox md-report` that generates comprehensive project reports covering stats, stories, commands, tests, dependencies, and contributors.

## Agent

@dev (Dex)

## Status

InReview

## Story

**As a** developer using AIOX,
**I want** to generate a comprehensive markdown report of my project,
**so that** I can share project status and metrics with stakeholders in a readable format.

## Acceptance Criteria

- [x] `aiox md-report` generates full markdown report to stdout
- [x] `aiox md-report --sections "stats,stories,commands,tests"` selects specific sections
- [x] `aiox md-report --output PROJECT-REPORT.md` writes to file
- [x] `aiox md-report --format json` outputs as JSON
- [x] Section: Project Overview (name, version, JS file count)
- [x] Section: Sprint Summary (story table with ID, title, status)
- [x] Section: Command Inventory (all registered CLI commands)
- [x] Section: Test Coverage (test file count, test directories)
- [x] Section: Dependencies (production and development deps)
- [x] Section: Contributors (from git log)
- [x] Zero external npm dependencies
- [x] Command registered in bin/aiox.js
- [x] 15+ tests in tests/cli/md-report.test.js

## Scope

**IN:**
- 6 report sections
- Section filtering
- Markdown and JSON output
- File output option

**OUT:**
- PDF generation
- Chart embedding
- Historical comparison
