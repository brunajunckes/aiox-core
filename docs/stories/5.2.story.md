# Story 5.2: Changelog & Release Notes Generator

## Objective

Implement `aiox changelog` to auto-generate a structured changelog from the git commit history of the project. Groups commits by conventional commit type (feat, fix, docs, chore, etc.) and outputs markdown ready to paste into a GitHub Release. Supports filtering by version tag or date range.

## Agent

@dev (Dex)

## Status

Draft

## Story

**As a** maintainer preparing a release,
**I want** `aiox changelog` to generate formatted release notes from git history,
**so that** I can produce a consistent, professional changelog in seconds instead of manually curating commit messages.

## Acceptance Criteria

- [ ] `aiox changelog` generates a markdown changelog grouped by conventional commit type from HEAD back to the previous version tag
- [ ] `aiox changelog --since v5.0.3` generates the changelog from the specified tag to HEAD
- [ ] `aiox changelog --since 2026-03-01` accepts ISO date as an alternative filter
- [ ] Output groups are: Features, Bug Fixes, Documentation, Chores, Other (for non-conventional commits)
- [ ] Each entry shows: commit subject line and short SHA (e.g., `- feat: add palette support (a1b2c3d)`)
- [ ] Commits that do not follow conventional commit format appear under "Other" — they are never silently dropped
- [ ] `aiox changelog --output CHANGELOG.md` appends the generated section to the specified file instead of stdout
- [ ] `aiox changelog --json` outputs machine-readable JSON instead of markdown
- [ ] Generated markdown is valid and renders correctly on GitHub
- [ ] Unit tests cover: commit parsing, type grouping, since-tag filter, since-date filter, JSON output

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
`aiox changelog` is a pure CLI tool. Output goes to stdout (default) or a file. No UI interaction.

**Zero External Dependencies:**
Use `child_process.execSync` to call git. Use only Node.js built-ins. No semver, no conventional-changelog npm packages.

**Git Log Command Pattern:**
```javascript
// All commits since tag
const range = sinceTag ? `${sinceTag}..HEAD` : 'HEAD';
const log = execSync(`git log ${range} --pretty=format:"%H %s"`, { encoding: 'utf8' });
```

**Commit Parser Pattern:**
```javascript
const CONVENTIONAL_RE = /^(feat|fix|docs|chore|refactor|test|perf|ci|style|build)(\(.+\))?!?: (.+)/;
function parseCommit(subject) {
  const m = subject.match(CONVENTIONAL_RE);
  return m ? { type: m[1], scope: m[2], message: m[3] } : { type: 'other', message: subject };
}
```

**File Append Pattern (`--output`):**
```javascript
const existing = fs.existsSync(outputPath) ? fs.readFileSync(outputPath, 'utf8') : '';
fs.writeFileSync(outputPath, newSection + '\n\n' + existing);
```

**Testing:**
- Mock `execSync` to return fixed git log output — no real git calls in unit tests
- Test location: `tests/changelog/` or matching existing test directory convention
