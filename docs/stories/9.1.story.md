## Dev Notes

**Architecture Constraint — CLI First (Article I):**
E2E tests validate the CLI interface, not internal modules. Tests must execute the actual `bin/aiox.js` binary via `execSync` to catch real issues.

**Test Location:** `tests/e2e/`

**Critical Patterns:**
```javascript
// Helper: Execute aiox command with captured output
function execAiox(command, options = {}) {
  const { cwd = process.cwd(), expectError = false } = options;
  try {
    const stdout = execSync(`node bin/aiox.js ${command}`, {
      cwd,
      encoding: 'utf8',
      stdio: 'pipe'
    });
    return { code: 0, stdout, stderr: '' };
  } catch (error) {
    if (expectError) {
      return { code: error.status || 1, stdout: error.stdout || '', stderr: error.stderr || '' };
    }
    throw error;
  }
}

// Helper: Run test in isolated temp directory
function withTempDir(testFn) {
  return async () => {
    const tempDir = fs.mkdtempSync(path.join(os.tmpdir(), 'aiox-e2e-'));
    try {
      // Copy necessary files to temp dir: bin/, .aiox-core/, package.json
      fs.cpSync(path.join(__dirname, '../../bin'), path.join(tempDir, 'bin'), { recursive: true });
      fs.cpSync(path.join(__dirname, '../../.aiox-core'), path.join(tempDir, '.aiox-core'), { recursive: true });
      fs.copyFileSync(path.join(__dirname, '../../package.json'), path.join(tempDir, 'package.json'));
      
      await testFn(tempDir);
    } finally {
      fs.rmSync(tempDir, { recursive: true, force: true });
    }
  };
}
```

**Relevant Source Tree:**
- CLI entry point: `bin/aiox.js`
- Command handlers: `packages/cli/commands/`
- Test config: `jest.config.js`
- Package scripts: `package.json`

**Testing Philosophy:**
- No npm package dependencies for E2E tests — only Node.js built-ins
- Temp directory isolation prevents test pollution
- Copy minimal files (bin/, .aiox-core/, package.json) to temp dir for realistic execution
- Each test is independent; can run in any order
- 30s timeout per test enforces CLI response performance

**Fixture Preparation:**
- Use `fs.mkdtempSync` for isolated test environments
- Minimal fixture setup: only what each test needs
- Use `process.cwd()` and relative paths, not hardcoded `/root/`
- Clean up with `fs.rmSync(tempDir, { recursive: true, force: true })`
