# Story 6.1: CLI Auto-Complete & Shell Integration

## Objective

Generate and install shell completion scripts for bash, zsh, and fish so that users can tab-complete all AIOX CLI subcommands, flags, and agent names. Running `aiox completion bash` outputs a ready-to-source bash script; `aiox completion install` auto-detects the current shell and appends the correct sourcing line to the user's rc file.

## Story

**As a** developer using the AIOX CLI daily,
**I want** tab-completion for all commands, flags, and agent names in my shell,
**so that** I can discover and invoke AIOX commands faster without memorising the full command surface.

- [ ] `aiox completion bash` prints a valid bash completion script to stdout
- [ ] `aiox completion zsh` prints a valid zsh completion script to stdout
- [ ] `aiox completion fish` prints a valid fish completion script to stdout
- [ ] `aiox completion install` detects the current shell (`$SHELL`), writes the completion script to the appropriate location, and appends the sourcing line to the user's rc file (`.bashrc`, `.zshrc`, or `~/.config/fish/completions/`)

- [ ] Completions cover all flags for each subcommand (e.g. `--verbose`, `--bail`, `--json`, `--sprint`)
- [ ] Completions include all agent names for commands that accept an agent argument (e.g. `aiox agents <TAB>`)
- [ ] `aiox completion install --dry-run` prints what would be written without making any changes
- [ ] Zero external npm dependencies — implemented with Node.js built-ins only
- [ ] Unit tests cover: script generation output for each shell, dry-run flag behaviour, shell detection logic

## Dev Notes

**Architecture Constraint — CLI First (Article I):**
Completion is a CLI-only feature. No UI surface needed. All logic lives under `packages/cli/completion/`.

**Zero External Dependencies:**
Use only Node.js built-ins: `fs`, `os`, `path`, `process`. No new npm packages.

**Command Manifest Source:**
Do not hardcode the command list. Read it from the same module that powers the CLI router so completions stay in sync automatically. If the router exports a `COMMANDS` map or similar, import it directly.

**Bash Completion Pattern:**
```bash
_aiox_completion() {
  local cur="${COMP_WORDS[COMP_CWORD]}"
  local commands="agents palette help resume status telemetry experiment feedback squads smoke-test changelog health docs doctor info validate install update config workers quickstart graph completion progress hooks release"
  COMPREPLY=( $(compgen -W "${commands}" -- "${cur}") )
}
complete -F _aiox_completion aiox
```

**Idempotency Guard:**
Before appending to rc file, check if the sourcing line already exists:
```javascript
const rcContent = fs.existsSync(rcPath) ? fs.readFileSync(rcPath, 'utf8') : '';
if (!rcContent.includes(sourceMarker)) {
  fs.appendFileSync(rcPath, `\n${sourceMarker}\n`);
}
```

**Testing:**
- Test location: `tests/unit/completion/`
- Mock `fs.writeFileSync` and `fs.appendFileSync` in unit tests — do not write to the real home directory
- Snapshot-test the generated scripts for bash, zsh, and fish

**Relevant Source Tree:**
- CLI router: `bin/aiox.js` (subcommand dispatch)
- Existing command modules: `packages/cli/commands/`
- Agent list: `.aiox-core/data/entity-registry.yaml`
