# Story 16.3: Logger with Levels

## Objective

Implement a leveled logger with file persistence via `aiox logger` that supports multiple log levels, metadata attachment, and JSONL-formatted file storage.

## Agent

@dev (Dex)

## Status

InProgress

## Story

**As a** developer using the AIOX CLI,
**I want** a structured logger with multiple levels,
**so that** I can debug and monitor applications with flexible log filtering.

## Acceptance Criteria

- [x] Logger module with 4 levels: debug (0), info (1), warn (2), error (3)
- [x] createLogger(name, options) factory with configurable minLevel
- [x] Convenience methods: debug(), info(), warn(), error()
- [x] log(level, message, meta) generic method
- [x] setLevel(level) to change minimum log level
- [x] getLogs(filter) with optional level, since, limit filters
- [x] clearLogs() to clear in-memory logs
- [x] Logs persisted to .aiox/logs/{name}.log in JSONL format
- [x] Metadata object support for each log entry
- [x] Automatic file rotation when file exceeds 10MB
- [x] `aiox logger show` displays logs with color-coded levels
- [x] `aiox logger show --logger <name>` filters by logger name
- [x] `aiox logger show --level <level>` filters by log level
- [x] `aiox logger show --filter <pattern>` regex filters messages
- [x] `aiox logger clear` removes log files with confirmation
- [x] `aiox logger clear --logger <name>` clears specific logger
- [x] `aiox logger clear --force` skips confirmation
- [x] `aiox logger clear --dry-run` previews deletion
- [x] 25+ tests in `.aiox-core/core/logger/__tests__/logger.test.js`
- [x] Command registered in `bin/aiox.js`
