# Story 4.3 — Monitor & Alerting

**Epic:** AutoFlow Phase 1  
**Status:** DONE ✅  
**Date Completed:** April 10, 2026

---

## Description

Implement resource monitoring (RAM, CPU, disk), container health checks, Prometheus metrics collection, and Grafana dashboards for observability.

---

## Acceptance Criteria

- [x] RAM/CPU/disk monitoring with auto-eviction when thresholds exceeded
- [x] Container health checks (60s interval, 5 consecutive failures = restart)
- [x] Prometheus metrics exported on `/metrics` endpoint
- [x] Grafana dashboards (read-only observability, no control)
- [x] Alert rules for critical thresholds (email to ops)
- [x] Systemd service integration (auto-restart on failure)
- [x] Lightweight monitoring (<5% CPU overhead)
- [x] Historical metrics stored (7-day retention)
