## Description

Implement 3-tier validation engine: Pydantic (deterministic) → Heuristic (rules) → LLM (semantic). Ensures all LLM outputs are safe, correct, and coherent before returning to user.

---

## Acceptance Criteria

- [x] Pydantic validation layer validates against schema (deterministic)
- [x] Heuristic validation layer applies business rules (rule-based checks)
- [x] LLM validation layer performs semantic validation (re-reads output)
- [x] Fallback on each tier (tier 2 if tier 1 fails, tier 3 if tier 2 fails)
- [x] Error taxonomy distinguishes hallucination vs constraint violation vs format error
- [x] 4.9K lines of test coverage (pydantic, heuristic, LLM tiers)
- [x] Performance < 500ms for full validation chain
- [x] Logging tracks which tier caught each error

## Technical Details

### 3-Tier Flow
```
Output → Pydantic Check → Heuristic Check → LLM Check → Response
        (schema)         (rules)           (semantic)
         ↓ FAIL           ↓ FAIL            ↓ FAIL
        Retry with       Retry with        Return error
        corrected        corrected
```

### Error Categories
- E2: Hallucination (detected by LLM validator)
- Constraint Violation (detected by heuristic validator)
- Format Error (detected by Pydantic validator)

### Performance Profile
- Pydantic: < 50ms
- Heuristic: < 100ms
- LLM: < 300ms
- **Total: < 500ms average**
