## Tenant Isolation Strategy

### PostgreSQL Schemas
```sql
CREATE SCHEMA tenant_001;
CREATE SCHEMA tenant_002;
CREATE SCHEMA tenant_003;
-- Each tenant has isolated tables: workflows, cache, costs, logs
```

### RLS Policies
```sql
CREATE POLICY tenant_isolation ON workflows
  USING (tenant_id = current_setting('app.tenant_id'));
```

### Context Injection
Every request:
1. JWT extracted → tenant_id
2. `SET app.tenant_id = 'tenant_001'` (PostgreSQL session variable)
3. All SQL queries use RLS (transparent to application code)
